# 요구사항에 맞추어 개발하는 커맨드

이 커맨드는 작성된 `./context/statement.md` 파일을 보고 작성된 구현 단계에 맞추어 실제로 구현을 진행합니다.

## 이 커맨트가 하는 일

1. **구현 단계에 맞추어 구현** - 구현 단계별 적절한 에이전트를 호출하여 구현.

- TDD :
  - 유틸 함수를 작성해야하는 경우
  - `tdd-guide.md` 문서 참고
- 구현 :
  - 컴포넌트를 구현하거나 훅을 구현해야하는 경우
  - `senior-frontend-implementer` 문서 참고
- 구현 외 작업 :
  - `senior-frontend-implementer` 문서 참고
- 코드 리뷰 : `code-reviewer.md` 문서 참고
- 리팩토링 : `refactor-cleaner.md` 문서 참고

2. **구현 완료마다 문서 업데이트** - 구현 단계를 완료할 때마다 `./context/statement.md`문서의 구현 단계를 업데이트.
3. **구현 완료마다 주의사항 검사** - 구현 단계를 완료할 때마다 `./context/statement.md`문서의 주의사항을 검사 후 수정.

## 작동 방식

- `./context/statement.md` 파일을 확인해 요구사항에 맞추어 에이전트를 호출해 구현을 진행합니다.
- 구현 단계를 하나씩 완료할 때마다 문서의 구현 단계를 업데이트합니다.
  - 업데이트를 할 때 진행한 구현 단계에 체크표시를 남깁니다
    e.g. `- [x]`
- 주기적으로 문서의 주의사항을 검사 후 수정이 필요하다면 수정합니다.

## 주의사항

- 구현시 사용하는 기술스택, 적용되는 코드 컨밴션 등에 위배되지 않아야 합니다.
  - 기술 스택은 CLAUDE.md 파일을 확인하세요
  - 코드 컨밴션에 적절하게 사용되었는지를 확인하기 위해 아래의 파일들을 확인하세요
    - `component-abstract-pattern` : 컴포넌트 추상화 레벨
  - 구현 중 서비스의 도메인 관련 지식이 필요하다면 `domain-knowledge`를 참고하세요
- 각 단계가 끝날 때마다 돌아가고있는 bashes는 kill 하세요
