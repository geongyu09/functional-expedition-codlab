# 리팩토링 대상 요구사항 작성 

이 문서는 리팩토링 대상 코드의 기능의 요구사항을 분석하도록 합니다. 
리팩토링 전 리팩토링 대상의 기능 요구사항을 명확히 이해할 수 있도록 도와줍니다. 
리팩토링 계획을 수립하기 이전에 올바른 요구사항을 뽑아내는데 사용합니다.

## 이 명령이 하는 일

- 입력받은 리팩토링 대상 기능을 분석합니다.
- 리팩토링 대상 기능의 요구사항을 ./context/requirement.md파일에 명확히 작성합니다.

## 입력

`./context/refactor-target.md` 파일에 리팩토링 대상 기능에 대한 정보가 담겨있습니다.
이 파일을 읽어 리팩토링 대상 기능을 파악합니다.

- 리팩토링 대상 기능에 대한 간단한 설명
- 리팩토링 대상 기능이 포함된 루트 파일 경로
- 리팩토링 범위 및 주의사항 등 추가 정보

### 입력 파일 예시

```markdown
# 리팩토링 대상 기능: 회원가입 페이지

- 루트 파일 경로: src/components/feature/pages/signup/index.tsx
- 설명: 사용자가 이메일, 비밀번호, 닉네임을 입력하여 회원가입할 수 있는 페이지입니다. 
- 주의 사항
  - 로그인 버튼 수정시 다른 페이지에 영향을 줄 수 있음
  - 반응형 디자인이 적용돼있음
```

## 작동 방식

- `./context/refactor-target.md` 파일을 읽어 리팩토링 대상을 확인합니다. 
- 리팩토링 대상의 코드를 확인합니다. 해당 코드에 영향을 미치는 코드 뿐만 아니라 해당 코드에게 영향을 주는 의존성 코드들도 함께 분석합니다.
- 리팩토링 대상 기능의 요구사항을 명확히 파악합니다.
- 코드가 수행하는 기능을 바탕으로 리팩토링 요구사항을 도출합니다. 
  - 타 에이전트가 해당 요구사항을 보았을 때 해야하는 동작이 리팩토링임을 명확히 이해할 수 있도록 작성합니다.
  - 리팩토링 대상 기능이 어떻게 동작하는지, 어떤 요구사항을 충족하는지 구체적으로 작성합니다.
  - 리팩토링 방향에 대한 제안도 포함할 수 있습니다.
- 도출한 요구사항을 `./context/requirement.md` 파일에 작성합니다.

### 결과 예시

```markdown
# 리팩토링 대상 기능 요구사항: 회원가입 페이지

회원가입 페이지를 리팩토링합니다.



## 주의사항 
- 요구사항은 구체적이고 명확하게 작성해야 합니다.
- 리팩토링 대상 기능의 모든 측면을 고려하여 요구사항을 도출해야 합니다.
- 요구사항 작성 시 코드의 현재 동작 방식을 정확히 반영해야 합니다.
- 리팩토링 방향에 대한 제안도 포함할 수 있습니다.
- 작성된 요구사항은 이후 리팩토링 계획 수립에 중요한 기준이 됩니다
