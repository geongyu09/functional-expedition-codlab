# pr 작성 커맨트

이 커맨트는 HEAD의 커밋과 develop 브랜치의 커밋의 변경사항을 파악하여 pr 내용을 작성합니다.

## 이 커맨트가 하는 일

1. **변경사항 확인** - 코드의 변경사항을 명확히 확인

- `git diff develop..HEAD` 명령어를 호출하여 변경된 코드 확인
- `git log develop..feature` 명령어를 호출하여 쌓인 커밋 메시지 확인
- `./context/statement.md` 파일을 확인하여 진행한 코드 변경사항 확인 및 주의사항 확인

2. **적절한 pr 형식에 맞추어 pr 문서 작성** - 확인된 변경사항 및 커밋 내용을 전체적으로 확인하여 pr 문서 작성

- `./context/pr.md` 파일에 pr 문서 작성

## 작동 방식

- 변경사항을 명확하고 정확하게 파악합니다.
  - `git diff develop..HEAD` 명령어를 호출하여 변경된 코드 확인
  - `git log develop..feature` 명령어를 호출하여 쌓인 커밋 메시지 확인
  - `./context/statement.md` 파일을 확인하여 진행한 코드 변경사항 확인 및 주의사항 확인

- 확인된 변경사항 및 커밋 내용을 전체적으로 확인하여 `./context/pr.md` 파일에 pr 문서를 작성합니다.
  - 해당 파일이 존재하지 않는경우 새롭게 만듧니다.
  - 해당 파일이 이미 존재하는 경우 내용을 덮어씁니다.

## pr 문서 형식

```
- #[이슈번호]

*주요 변경사항 및 pr 내용 요약(초록)내용 작성*
한 문장에서 두 문장으로 작성. **단 높임말 사용.**

## 주요 변경사항

### [변경사항 1]
만약 변경 사항이 많다면 개괄식이 아닌 h3로 하나씩 나누어서 줄글로 설명
설명시에는 ~하였습니다. 방식의 **높임말 사용**

## 리뷰 주의사항

- 리뷰어에게 중점적으로 봐주길 원하는 부분 개괄적으로 작성
- 필요 없다면 생략 가능
- **단 높임말 사용.**

```

## pr 문서 작성 예시

```md
- #51

## 주요 변경 사항

프로필 사진 업로드 및 제거 기능을 구현하였습니다.

### 프로필 사진 업로드

프로필 사진의 경우 s3를 통하여 관리하도록 하였습니다.
이에 따라서 presignedUrl을 받은 이후, 프론트에서 파일을 업로드 하고 올바르게 업로드가 되었다는 메시지를 백엔드에게 보내는 방식입니다.
여기에서 업로드라는 동작 하나에 3개의 api 호출을 해야합니다.
현재로서는 3개의 api를 하나의 비동기 함수로 만들고, 이를 핸들러에서 관리하도록 구현하였습니다.
다만 이는 기존의 api(axios) -> query hook -> components 의 계층을 이루던 컨밴션에 위배되는 코드이므로, 추후 개선하도록 하겠습니다.

### 업로드할 프로필 사진 수정

요구사항에 따라서 업로드할 프로필의 크기 및 위치를 수정할 수 있어야 했습니다.
이에 따라서 GestureDetector와 Animated.Image를 통하여 이미지를 이동할 수 있도록 하였습니다.
이후, 이동한 위치 및 확대한 크기를 계산하고, image-manipulator를 사용하여 해당 위치에 해당 크기만큼 crop하도록 하였습니다.
```
